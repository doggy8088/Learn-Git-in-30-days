# Day 19: Setting .gitignore Ignore List

Not all files in your project directory should be tracked by Git. Files like build artifacts, temporary files, and sensitive configuration files should be ignored. Git provides the `.gitignore` file for this purpose.

## Understanding .gitignore

A `.gitignore` file specifies intentionally untracked files that Git should ignore. Files already tracked by Git are not affected by `.gitignore`.

## Creating .gitignore

Simply create a file named `.gitignore` in your repository root:

```
# On Windows
echo node_modules/ > .gitignore

# On Linux/Mac
echo "node_modules/" > .gitignore
```

Or create it manually with a text editor.

## Basic Patterns

### Ignore Specific Files

```
config.local.js
secrets.txt
```

### Ignore Files with Specific Extensions

```
*.log
*.tmp
*.cache
```

### Ignore Directories

```
node_modules/
dist/
build/
```

Note the trailing slash to indicate a directory.

### Ignore Files in Any Directory

```
# Ignore all .log files in any directory
**/*.log
```

## Advanced Patterns

### Negation (Exception)

Use `!` to make exceptions:

```
# Ignore all .log files
*.log

# But don't ignore important.log
!important.log
```

### Ignore Files in Specific Directory Only

```
# Ignore test.txt only in root directory
/test.txt

# Ignore test.txt in any subdirectory, but not root
*/test.txt
```

### Multiple Extensions

```
# Ignore multiple file types
*.{jpg,png,gif}
```

## Common .gitignore Patterns

### Node.js Projects

```
node_modules/
npm-debug.log
.env
.DS_Store
dist/
build/
```

### Python Projects

```
__pycache__/
*.py[cod]
*$py.class
.venv/
venv/
*.egg-info/
.pytest_cache/
```

### Java Projects

```
*.class
*.jar
*.war
target/
.gradle/
build/
```

### .NET Projects

```
bin/
obj/
*.user
*.suo
packages/
.vs/
```

## Global .gitignore

For patterns you want to ignore across all repositories:

```
git config --global core.excludesfile ~/.gitignore_global
```

Then create `~/.gitignore_global` with your patterns.

## Ignoring Already Tracked Files

If you accidentally committed files that should be ignored:

```
# Remove from Git but keep the file
git rm --cached filename

# Remove directory from Git but keep it
git rm -r --cached directory/
```

Then add the pattern to `.gitignore` and commit.

## Viewing Ignored Files

To see which files are being ignored:

```
git status --ignored
```

## Checking Why a File is Ignored

To see which pattern is causing a file to be ignored:

```
git check-ignore -v filename
```

## Template .gitignore Files

GitHub provides template .gitignore files for various languages and frameworks:

* https://github.com/github/gitignore

## Comments in .gitignore

Use `#` for comments:

```
# This is a comment
*.log  # Inline comment
```

## Best Practices

1. **Commit .gitignore early**: Add it at the start of your project
2. **Be specific**: Avoid overly broad patterns
3. **Document why**: Use comments to explain non-obvious patterns
4. **Keep it organized**: Group related patterns together
5. **Use templates**: Start with language-specific templates
6. **Local variations**: Use `.git/info/exclude` for personal patterns

## Subdirectory .gitignore

You can have `.gitignore` files in subdirectories. Patterns are relative to the location of the `.gitignore` file.

## Today's Summary

The `.gitignore` file is essential for keeping your repository clean and avoiding accidentally committing files that don't belong in version control. Understanding the pattern syntax and best practices will help you maintain a clean, professional repository.
